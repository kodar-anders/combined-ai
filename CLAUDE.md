## What this is

A **plain TypeScript library** that combines several AI providers into one package — pick one provider for a prompt, or combine many via a strategy. `src/index.ts` is the main public entry point; re-export the public API there (provider classes stay internal). A second `combined-ai/test` subpath entry (`src/testing/index.ts`) exports the network-free `MockProvider` (+ re-exports `ProviderError` so `instanceof` holds across the separately-bundled entry) — kept off the main entry so it never lands in a production bundle. Three providers: Anthropic, OpenAI, Google. The `"google"` provider speaks the **Gemini** API — registry key/`name` is `"google"` (the company, like `anthropic`/`openai`); internal `toGemini*` helpers and the `gemini-3.5-flash` default keep the Gemini name (the API it talks to).

## Core contracts — `src/types.ts`

The provider-agnostic contract: `Message`, `CompletionRequest`, `CompletionResult`, `Provider` (`complete()` → text, `stream()` → `AsyncIterable<string>` deltas, **optional** `embed?()` → `EmbeddingResult`). Points an agent can't infer:

- **Embeddings:** `embed?()` is **optional** on `Provider` (Anthropic has no first-party endpoint → omits it; OpenAI + Google implement it). Reach it via `ProviderRegistry.embed`/`embedMany` (the `#selectEmbedder` helper throws if unsupported). `EmbeddingResult.usage` **reuses `Usage`** with `outputTokens:0`, `totalTokens===inputTokens` so embedding calls price/aggregate through the existing cost machinery; embedding models are in `MODELS` (input-only, `outputPerMTok:0` — full exact keys, the digit-suffix resolver won't reach `-small`/`-large` from a base). Google's embed endpoint reports no usage → `usage` omitted. `cosineSimilarity` lives in `src/embeddings.ts` (pure; always cosine, never raw dot-product).

- `Message.content` is `string | ContentPart[]` (string = one text part). `ContentPart` union: text/image/file/`tool_use`/`tool_result`. `MediaSource` = base64 (carries mediaType) | url (optional mediaType). **Intended compile seam:** adding a `ContentPart` member breaks every provider `to*Part` mapper and `textOf` in `combine/shared.ts` until handled. **Provider support varies — mappers throw when unsupported.** Optional `cacheControl?: CacheControl` on text/image/file parts (and `system?: string | SystemPrompt` for the system prompt) is the Anthropic prompt-cache marker — see Cost & pricing → request-side caching.
- `complete()`-only result fields, all optional + set only when present (so `toEqual({text,model})` tests still pass): `finishReason` (normalized `stop|length|content_filter|tool_use|other`) + `rawFinishReason`; `refusal` (forces `content_filter`); `usage` (`{inputTokens,outputTokens,totalTokens}`); `parsed`; `toolCalls`. New code building a `CompletionResult` should keep these.
- `signal?: AbortSignal` forwards to every provider `fetch`; combine threads one signal through all participant calls. `retry?: RetryOptions` + `timeoutMs?: number` (also on `EmbeddingRequest`) are per-request transport overrides (see Errors & transport). **Semantic split:** `signal` bounds the whole run (one signal cancels a combine / a fallback chain), `timeoutMs` bounds **each individual provider call** (in combine/fallback that's per participant/entry, so worst-case wall-clock is `calls × timeoutMs`, not a run-wide deadline).
- **Structured output:** `responseFormat?: {type:"json_schema", schema, name?}` — raw JSON Schema (no Zod; locked choice). Affects `complete()` and `stream()`; `complete()` also returns `parsed` (= `JSON.parse(text)`, undefined on invalid/truncated). **Cross-provider schema rules** (driven by OpenAI strict + Gemini's limited keywords): every object needs `additionalProperties:false` + all props `required` with a single non-null type; avoid optional/nullable, recursive, `$ref`, numeric/length constraints (`toGeminiSchema` doesn't translate null-unions).
- **Tool calling (single-provider only):** `tools?` + `toolChoice?` (`auto|any|none|{name}`); replay results with `tool_use`/`tool_result` parts. **`combine` rejects tools/toolChoice.** Replay rules: OpenAI throws without `tool_use.id`/`tool_result.toolUseId`; Gemini throws without `tool_result.name`; Anthropic hoists `tool_result` blocks first.

## Errors & transport

- `src/errors.ts` — error vocabulary. `ProviderError` (exported), `kind`: `"api"` (non-2xx → `status` set, via `apiError`) | `"transport"` (`fetch` rejected / signal aborted → `cause` set, via `transportError` — the one place the transport-error shape lives; `transport.ts`, `sse.ts` (body-read aborts), `readJsonBody`, `apiError` (its error-body `text()` read), and the test `MockProvider` all throw through it). `apiErrorFromBody` handles a 2xx body carrying `{error}` (so it throws instead of returning `text:""`). **Imports `ProviderName` from `./registry` type-only** (no runtime cycle — keep it that way). Mid-stream SSE **error events** stay plain `Error` (only the `reader.read()` abort is wrapped). Also `aggregateError(message, causes)` (→ `AggregateError` or plain `Error`); combine's `noResultError` adapts it. No fetch/transport code here.
- `src/transport.ts` — `requestWithRetry(provider, input, init, retry?)`: bounded exponential backoff on 429/503/529, honors `Retry-After` (capped 60s), aborts early on `signal`. **Transport rejections aren't retried.** Config via per-provider `retry?: RetryOptions` (`{maxRetries?, baseDelayMs?}`, defaults 2/500ms, `0` disables). Per-`complete()` call, so combine benefits transparently. **Per-request overrides:** each provider calls `requestControls(request, this.#retry)` once per `complete()`/`stream()`/`embed()` → `{signal, retry}`. `mergeRetry` field-merges `request.retry` **over** the construction-time retry (spread, so explicit `maxRetries:0` wins); `withTimeout` folds `request.timeoutMs` into the signal via `AbortSignal.any([signal, AbortSignal.timeout(ms)])` — a **whole-call** deadline (all retries + backoff + stream body read), aborting with a `TimeoutError` (distinct from a caller abort). `assertValidTimeoutMs` (exported; also called up-front in the registry `#prepare`) rejects non-positive / non-finite / `> 2_147_483_647` (the `setTimeout` overflow footgun). **`RetryOptions` is defined in `types.ts`** (keeps that leaf a leaf) and **re-exported from `transport.ts`** for back-compat — don't move it back. `readJsonBody(provider, response)` wraps an abort/network failure **during** the `.json()` body read as a transport error (a `SyntaxError` from a malformed body stays raw); the 3 `complete()` sites use it since `requestWithRetry` only wraps up to the response headers.

## Cost & pricing — `src/models.ts` + `src/cost.ts`

Data/math split (mirrors errors/transport). Both re-exported from `index.ts`.

- `src/models.ts` — internal `MODELS` table (USD per MTok, **kept small on purpose**: the most common models per provider, not an exhaustive catalog — a wrong price is worse than `undefined`), `findModel(model, options?)`, `listModels()`, `PRICING_VERIFIED_ON`. **Resolution: exact → dated-snapshot prefix** — a key `k` matches `k-<DIGIT>…` only (the char after `k-` must be 0-9), so date/version snapshots resolve (`gpt-4.1-2025-…`→`gpt-4.1`, longest key wins) but **differently-priced siblings with a word suffix return `undefined`, never a wrong price** (`gpt-4.1-nano`, `gemini-2.5-flash-lite`; the digit rule is load-bearing — don't loosen to bare `startsWith(k+"-")`). Namespaced gateway ids (`openai/gpt-4.1`) correctly miss. `options.models` merges **over** built-ins (override wins); no merge alloc when no overrides. `ModelInfo.id` is the resolved key; returned `pricing` is a **copy** (`clonePricing`, incl. nested `highTier`) so callers can't corrupt the singleton. **Tiered pricing** via optional `ModelPricing.highTier` (`{aboveInputTokens, …}`) — Gemini 2.5 Pro doubles above 200k prompt tokens. **Cache pricing** via optional `cachedInputPerMTok` (read, **tier-aware** via `highTier.cachedInputPerMTok`) + `cacheWriteInputPerMTok` (write, base-only — Anthropic has no highTier); Anthropic carried (read 0.1×, write 1.25× = 5-min TTL, so 1-hour writes under-bill) and Gemini carried (read 0.1×, 2.5 Pro tiered), **OpenAI cache rates unset** (live page now lists only gpt-5.x → gpt-4.x cached rates unverifiable → fall back to full input rate). Prices are best-effort/hand-maintained — **a wrong price is worse than `undefined`**; keep the table small and update `PRICING_VERIFIED_ON` when touching prices. To refresh the table + per-provider defaults against the providers' live docs, run the **`/update-models`** skill (`.claude/skills/update-models/`).
- `src/cost.ts` — `costOf(result, options?)` / `costOfUsage(usage, model, options?)` → `CostBreakdown` (USD) or **`undefined`** (never throws): unknown model, no usage, or **`inputTokens <= 0`** (empty `{}`/malformed usage — a real completion always has prompt tokens; declining beats a confident `$0` or input mis-billed at the output rate). Picks `highTier` rates when `inputTokens > aboveInputTokens`. **Cache split:** `inputCost = uncached×inputRate + cachedInputTokens×(cachedRate ?? inputRate) + cacheCreationInputTokens×(writeRate ?? inputRate)`, where `uncached = max(0, inputTokens − cached − write)` (clamped so gateway over-reports can't go negative); folded into `inputCost` (no separate `CostBreakdown` field). `cachedRate` is read off the **selected tier** (so Gemini 2.5 Pro's cached rate doubles >200k); `writeRate` off base. **Gemini thinking tokens**: bill `outputTokens + max(0, totalTokens−input−output)` at the output rate (0 residual for Anthropic/OpenAI). **Never price off `totalTokens`** (cross-provider semantics differ). Float dollars — `(tokens*perMTok)/1e6`.
- **Prompt-cache accounting (the gotcha):** `Usage.inputTokens` is the **total billable prompt** (a superset incl. cache reads/writes); `cachedInputTokens`/`cacheCreationInputTokens` are **subsets** of it (set only when reported). Providers differ — Anthropic reports cache tokens in buckets **outside** `input_tokens` (so `extractUsage` sums them into the superset; defensively returns `inputTokens:0` if base `input_tokens` is missing, so cost declines rather than underbills), while OpenAI (`prompt_tokens_details.cached_tokens`) / Gemini (`cachedContentTokenCount`) report cached as a subset already inside their prompt count (clamped to `inputTokens`). **Reporting is `complete()`-only** — `stream()` reports no usage. OpenAI + Gemini 2.5 cache automatically; mark Anthropic breakpoints via `CacheControl` (below).
- **Request-side caching (Anthropic):** `CacheControl = {ttl?: "1h"}` on a `ContentPart` (`cacheControl` on text/image/file) or the system prompt's object form `system: {text, cacheControl}` (`SystemPrompt`). `anthropic.ts` emits `cache_control` on those blocks; `prepareCacheControl` enforces the **≤4 breakpoint** limit (throws a clear error, not a raw 400) and toggles the `extended-cache-ttl-2025-04-11` beta header **only** when a `ttl:"1h"` breakpoint is present (computed once, passed to `#headers` — don't re-scan). OpenAI/Gemini **ignore** the marker (read system `.text` only). **`combine` ignores `cacheControl`** — `composeSystem`/`completionFor` forward system text only (the union's `.text`), and its strategies rebuild prompts.
- `combineCost`/budget shipped in `src/combine/cost.ts` (roadmap #2) — cache savings flow through both automatically (per-call ledger priced via `costOfUsage`). See Combine section.
- Deferred (see `/tmp/plan-cost-pricing.md`): capability flags → later. Streaming usage, Gemini _explicit_ caching, and verified OpenAI cache-read rates still pending.

## Providers — `src/providers/`

All talk to their HTTP API directly over `fetch` — **no SDK dependency**. Each maps `Message.content` per-wire, normalizes `finishReason`/`usage`, supports structured output + tools. Class stays internal; `index.ts` re-exports only the options type. Gotchas:

- `anthropic.ts` — Messages API, default `claude-opus-4-8`. **Body kept minimal — Opus 4.x rejects `temperature`/`top_p`/`top_k`/`thinking`.** `refusal` collected from `type:"refusal"` blocks; hoists `tool_result` blocks first. **Prompt caching:** emits `cache_control` on marked text/image/document blocks + object-form `system`; `prepareCacheControl` caps breakpoints at 4 + gates the 1h beta header (see Cost & pricing).
- `openai.ts` — Chat Completions, default `gpt-5.6-terra` (the 5.6 family — Sol/Terra/Luna, GA 2026-07-09 — has no mini/nano tier). `system` folded into a leading system message; cap sent as `max_completion_tokens`. `OpenAIProviderOptions.headers?` spread last (override defaults) — for gateways/OpenAI-compatible. Constructor's 2nd arg `name` (internal, registry-only) attributes errors to a custom alias. **URL file source throws** (Chat Completions has none); tool-call `arguments` is a JSON string (parsed on the way out).
- `google.ts` — Gemini API (`GoogleProviderOptions`), default `gemini-3.5-flash` (the 2.5 generation retires 2026-10-16; `gemini-2.5-pro` already 404s for new keys). Model+action in the URL path; `assistant`→`model`; `system`→`systemInstruction`; cap→`maxOutputTokens`. **Thinking-token gotcha: Gemini 2.5/3.x thinking tokens count against `maxOutputTokens`, so a tiny cap returns empty/truncated text — give it a generous cap.** **`fileData.fileUri` accepts only a Files API/`gs://` URI, not a public web URL.** `toGeminiSchema` UPPERCASEs `type` and drops `additionalProperties` (Gemini 400s on it though OpenAI strict requires it) for structured output + tool params; finishReason overridden to `tool_use` only on a clean stop; no `refusal` field.
- Shared helpers: `extract.ts` (`extractModel`, exported `isRecord`), `structured.ts` (`parseStructured`), `sse.ts` (`sseJson(body, provider)` — shared SSE reader: yields parsed `data:` JSON objects, ends on `[DONE]`, always releases the reader (guarded `finally` so a cancel rejection can't mask the outcome); wraps a `reader.read()` abort/network failure as a transport error via `provider`; each `stream()` for-awaits it and handles its own events/errors). `toArray` stays duplicated per-provider (trivial). `errors.ts`/`transport.ts` keep their own `isRecord` (lower layer).
- **Adding a provider:** new `src/providers/<name>.ts` implementing `Provider`; add an optional field to `ProviderRegistryConfig` + a construction branch in the registry; re-export its options type from `index.ts`.

## Registry — `src/registry.ts`

`ProviderRegistry` is the **single point of access**; provider classes aren't exported. Constructor takes `ProviderRegistryConfig` (`{anthropic?, openai?, google?, custom?}`) and builds only what's present. **Never reads env** — keys come from config.

- `select(name)` → `Provider` or throws (lists configured names); `has(name)`, `names()` (built-ins first in fixed order, then custom in definition order).
- `custom?: Record<string, CustomProviderConfig>` — `{kind:"openai-compatible", apiKey, baseUrl, model, headers?, retry?}` (OpenRouter/Together/Groq/Ollama/local; `baseUrl` excludes the request path) | `{kind:"provider", provider}` (BYO). A custom name colliding with a built-in throws.
- `ProviderName = BuiltInProviderName | (string & Record<never,never>)` (autocomplete trick; `Record<never,never>` satisfies `no-empty-object-type`) — custom names accepted everywhere, built-ins still autocomplete.
- **Combine API:** four per-strategy methods (`consensus`/`pipeline`/`ensemble`/`broadcast`, each returns its concrete `…Result`) + generic `combine(request)` dispatching on `request.strategy ?? "consensus"`. `combine<S extends StrategyName>` infers `S` from the `strategy?: S` discriminant → concrete `ResultFor<S>` for a literal strategy, full union otherwise. (Don't type the param as `StrategyRequest<S>` — TS can't infer through indexed-access on object literals.)
- **Fallback (`src/fallback.ts`):** `fallback(specs, options?)` → a composable `Provider` (not a wire provider — it wraps resolved providers, like combine wraps a roster; imports `ProviderName` **type-only**, registry imports the factory at runtime). Tries entries in order, catching `ProviderError`. `FallbackSpec = ProviderName | {provider, model?, maxTokens?}` (overrides beat the per-call request, entry→request→default — mirrors `completionFor`). Abort (`request.signal.aborted`, since abort & network are both `kind:"transport"`) propagates without advancing; non-`ProviderError` propagates (a bug); all-fail throws `aggregateError` (≥1 cause even for 1 entry — push cause **before** the last-entry check). `stream()` falls back **only before the first delta** (commit after first `next()`; `try/finally { it.return?.() }` closes the inner stream on early `break`). Full request forwards unchanged (**unlike combine** — tools/responseFormat/cacheControl pass through). `shouldFallback`/`onFallback` both take one `FallbackEvent {provider, index, error}` (permissive default; `onFallback` fires N-1× on all-fail). **No `embed`.** No per-entry `retry` (that's #7's axis).
- Validation: shared `#prepare` normalizes each `ParticipantSpec` (throws on a falsy `model`/`maxTokens` override), enforces ≥1 participant + unique ids + non-empty messages, rejects tools/toolChoice, resolves the roster. Per-strategy: consensus/pipeline/broadcast reject `responseFormat` (ensemble-only); consensus validates `minParticipants`/`synthesizer`; ensemble requires `responseFormat` with an object root.

## Combine — `src/combine/`

Make multiple providers cooperate on one prompt via a selectable strategy. **One file per strategy.** Entry is `ProviderRegistry.combine`; each orchestrator is internal (the registry imports it). No streaming in combine yet.

- `index.ts` — public barrel: combine types + the one runtime value `STRATEGY_NAMES` (source of truth `combine()` validates). `CombineResult` discriminated on `strategy` (`BroadcastResult` has **no `text`**). `ParticipantSpec = ProviderName | {provider, model?, maxTokens?, label?}` with a unique **id** (label, defaults to provider or `<provider>-<model>`); `synthesizer` is an id. Results carry `usage?: CombineUsage` (`total`/`byParticipant` keyed by id + a per-call `calls: CallUsage[]` ledger, each tagged with `model` for individual pricing). **Imports `ProviderName` type-only** (keep the combine↔registry cycle erasable).
- `cost.ts` — `combineCost(result, options?)` → `CombineCost {totalCost, byParticipant}` (USD), pricing each `usage.calls` entry **individually** (never the summed `byParticipant` — summing breaks tiered rates / thinking residual); skips unpriceable calls, `undefined` if none priceable. Imports `costOfUsage` from `../cost` (one-way combine→cost; `cost.ts` stays single-result). `combineCost`/`CombineCost`/`CallUsage` re-exported from root `index.ts`.
- **Budget** (`CombineOptions.budget?: {usd} & CostOptions`) — soft floor on _optional_ work, not a hard cap. `makeBudget`→`BudgetTracker` (shared) tracks running cost via `costOfUsage`; strategies call `budget.gate(skipped, extra?)` (emits the skip event + returns whether to skip — keeps the event shape in one place, no-op tracker returns false so no `options?.budget` re-check needed). consensus skips critiques/sanitize, pipeline stops refining/skips sanitize. `exceeded()` is `spent > 0 && spent >= usd` (the `spent > 0` makes an unpriceable roster — or a `usd:0` budget — inert, not falsely tripped). **Required phases (consensus drafts+synth, pipeline first stage) always run** so a run never ends empty. Unpriceable call → contributes 0 + one-shot `{type:"budget", underEnforced:true}` event. ensemble/broadcast accept budget but it's **inert** (single parallel burst can't be gated → no `budget.add`, no event; price with `combineCost`). Gating is coarse — checked only between `Promise.all` phases. `budget` is a new `CombineEvent` variant (skip events carry `skipped`, pipeline-refine skip also `id`/`index`).
- `shared.ts` — helpers reused by ≥2 strategies (`completionFor`, `composeSystem`, `runOutcome`, `respondAll`, `noResultError`, `sanitizeAnswer` (returns `model` too), `makeEmitter`, `aggregateUsage`, `callUsages`, `makeBudget`/`BudgetTracker`, `RosterEntry`, …). `completionFor`/`sanitizeAnswer` take per-participant overrides that beat the request-wide `model`/`maxTokens`; strategies pass the entry itself. `completionFor` also forwards `retry`/`timeoutMs` (per participant call, unlike the run-wide `signal`). A helper lands here only once a 2nd strategy reuses it (2–3x rule).
- **consensus** — draft (parallel) → critique (each survivor critiques all drafts, anonymized unless `attribution:"attributed"`) → synthesize (by `synthesizer` id) + a sanitizing pass. **Framing invariants (preserve — rationale in doc-comments):** `SYNTH_FRAMING` adjudicates on correctness over popularity and forbids alluding to drafts/critiques/labels; `sanitizeAnswer` strips leftover process narration (one extra `complete()` call, multi-provider only); `CONCISE_DIRECTIVE` strips ceremony but keeps reasoning/caveats. Survivor policy: empty drafts dropped; 0 → throw; single-provider → plain completion; `< minParticipants` (default 2) → throw; failed critiques non-fatal; failed synthesizer falls back to next survivor.
- **pipeline** — sequential refinement; participant order = conveyor order. First stage drafts, later stages refine the running answer; a stage advances only if `ok` + non-empty; final = last advancing stage. Throws only if none advanced. Sanitize runs only when a refiner actually changed the text. No `minParticipants`/`synthesizer`/`attribution`.
- **ensemble** — multi-model vote on structured output (the differentiator). Every participant answers under the same `responseFormat` in parallel; only `ok` responses with a plain-object `parsed` count. **Mechanical merge (no LLM):** field-wise majority vote over the first-seen key union (mode by deep-equality, first-seen wins ties) so merged values are real. `agreement` denominator = all valid responses (omitted fields score low). Object-root only; nested merge is shallow. (`stableKey` uses `.sort()` not `toSorted` — ES2022 lib lacks it.)
- **broadcast** — fan-out, **no combine**: every participant answers the verbatim prompt in parallel (no framing/sanitize), returns all responses, no `text`. Throws only if zero succeed (empty `ok` still counts). Rejects `responseFormat`.
- **Embedding signals** (`CombineOptions.embedding` = `{provider, model?}`, resolved in the registry `#resolveEmbedder`→`#selectEmbedder`, passed to the orchestrator as a trailing `embedder?` arg). All **informational** (never change a returned/merged value), usage folds into the ledger under a distinct `embedding:<provider>` id (so it can't collide with a participant id), failure is swallowed, and a provider returning a mismatched vector count makes the helper decline: **broadcast** `result.semantic` (`agreement`/`outlier`/`clusters`); **consensus** `result.draftAgreement` (same `SemanticComparison` over surviving drafts, computed concurrently with critique/synth); **ensemble** `result.semanticAgreement` (per-field mean-pairwise-cosine over **string** fields only, one batched embed call — exact-match vote still picks `merged`). **pipeline** unaffected (no parallel answers).
- `embedding.ts` — combine's embedding module (like `cost.ts`), used by broadcast/consensus/ensemble: `compareAnswers` (whole-answer → `SemanticComparison`), `fieldSemanticAgreement` (per-field string scores), `ResolvedEmbedder`. `agreement`=mean pairwise cosine, `outlier`=centroid distance (≥3), `clusters`=seed-based (threshold const, deterministic). Principle: **embeddings in combine rank/inform, never decide a value.**
- **Test coupling (gotcha):** consensus/pipeline tests classify the phase by literal markers in the framing constants — `"Rewrite the following"` (sanitize), `"lead assistant"` (synth), `"produce only a critique"` (critique), `"revise the current answer"` (pipeline refine). Keep each constant free of the others' markers.
- **`noUncheckedIndexedAccess` gotcha:** don't zip parallel arrays by index — carry paired data in one structure (roster / running answer).
- **Adding a strategy:** new `src/combine/<strategy>.ts` exporting its orchestrator; add to `STRATEGY_NAMES`/`StrategyName`, a `<Strategy>Request`, a `…Result` member, and a `StrategyRequest` entry in `index.ts`; add a public method + a `combine()` `case` in the registry. Promote a helper to `shared.ts` only on actual reuse.

## Toolchain & conventions

- **Build = tsup, type-check = tsc** (`noEmit: true`). Tests run through **`@swc/jest`** (no type-checking — that's `yarn typecheck`); test files import globals from `@jest/globals`. - **Yarn 4 (Berry) PnP** — no `node_modules`; always `yarn`, never `npm`. **PnP gotcha:** can only import deps declared directly in `package.json` (why `@jest/globals` is an explicit devDep). `@jest/globals` version lags `jest` — match what Jest pulls. `tsconfig` sets `ignoreDeprecations:"6.0"` for tsup's `.d.ts` `baseUrl`.
- **Lint** (`eslint.config.mjs`, flat, type-aware + unicorn + promise; Jest rules in tests only): `type` over `interface`; inline `import type` + `consistent-type-exports`; exported API needs explicit types/return types; `no-console: error` (off in tests); `_`-prefix unused args; single-line `case` omits braces; Prettier owns formatting.
- **Publishing:** push a `v*` tag → `.github/workflows/publish.yml` verifies tag==version, runs checks, publishes to npm via OIDC trusted publishing (no token), and creates a GitHub release from the matching `CHANGELOG.md` section.
